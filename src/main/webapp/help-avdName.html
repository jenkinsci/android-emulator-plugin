The names of existing Android emulator configurations (AVDs) can be seen by running the following command from the Android SDK: android list avd. Remote running devices can also be accessed in this manner and are typically named as host:port

For named emulators the AVD must already exist at build time on the machine the build will be executed on — the emulator configuration will not be automatically created or copied from another machine.

You can also do environment variable substitution for this setting, in the format: ${VARIABLE_NAME} or $VARIABLE_NAME.
